# 数据看板与复盘模板（30 天试运行）

## 1) 主数据表结构（ContentItem）

```ts
type ContentItem = {
  id: string;                  // 例：2026W1-V1
  week: number;                // 1~4
  pillar: 'concept' | 'framework' | 'checklist' | 'case';
  format: 'video' | 'image-text';
  title: string;
  hook: string;                // 前3秒钩子
  cta: string;                 // 站内动作
  targetAudience: 'newbie' | 'pro' | 'mixed';
  publishAt: string;           // ISO datetime
  status: 'draft' | 'published' | 'rework';
};
```

## 2) 表现快照结构（PerformanceSnapshot）

```ts
type PerformanceSnapshot = {
  contentId: string;
  impressions: number;
  plays?: number;
  likes: number;
  comments: number;
  saves: number;
  follows: number;
  ctrCover?: number;
  retention3s?: number;
  completionRate?: number;
  engagementRate: number;      // (likes+comments+saves)/impressions
  followConversionRate: number;// follows/plays
};
```

## 3) 周复盘结构（WeeklyReview）

```ts
type WeeklyReview = {
  week: number;
  bestPerformerIds: string[];
  worstPerformerIds: string[];
  winningPatterns: string[];
  failedPatterns: string[];
  nextWeekAdjustments: string[];
};
```

## 4) 每周复盘 SOP（周日执行）

1. 拉取当周 5 条内容核心数据（曝光、播放、赞评藏、关注、留存、完播）。
2. 选出 Top2 与 Bottom2 内容，做“结构拆解”：
   - 标题是否结果导向？
   - 首屏是否 3 秒结论？
   - CTA 是否明确且站内？
3. 输出 3 条下周必须执行的变更：
   - 一条改标题模板
   - 一条改开头模板
   - 一条改互动引导
4. 更新“选题池状态”：保留 / 重做 / 下线。
